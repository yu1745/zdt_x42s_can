# ZDT_X42S CAN 分帧规则（手册 4.2.1，真机验证）

> 本文档记录 ZDT_X42S 第二代闭环步进电机手册第 4.2.1 节"主机发送命令格式说明"
> 的 CAN 分帧规则。权威 PDF 在 libzdt 仓库：
> [`ZDT_X42S第二代闭环步进电机用户手册V1.0.3_251224.pdf`](https://github.com/yu1745/libzdt/blob/master/ZDT_X42S%E7%AC%AC%E4%BA%8C%E4%BB%A3%E9%97%AD%E7%8E%AF%E6%AD%A5%E8%BF%9B%E7%94%B5%E6%9C%BA%E7%94%A8%E6%88%B7%E6%89%8B%E5%86%8CV1.0.3_251224.pdf)

## PDF 原文（4.2.1，第 2195 行附近）

> 1. CAN 通讯帧类型固定为 **扩展帧格式**；
> 2. Addr 为电机地址，Packet 为第几包数据，从 0 开始计数。对于小于 8 字节数据的命令，
>    Packet 为 0，例如，命令"01 36 6B"，则将扩展帧 ID 设置为 **0100**，发送数据内容为 **36 6B**；
>    对于大于 8 字节命令，需要进行拆包发送，Packet 从 0 开始计数，
>    例如，命令"01 FD 01 0F A0 00 00 01 FA 00 00 00 6B"，拆分为：
>    第一条发送：扩展帧 ID 设置为 **0100**，数据内容为 **FD 01 0F A0 00 00 01 FA**，
>    第二条发送：扩展帧 ID 设置为 **0101**，数据内容为 **00 00 00 6B**；

**手册原文是对的**。本仓库早期版本曾误以为有勘误（"每包首字节为地址"），
真机测试推翻了它，最终规则与手册原文一致。

## 正确规则（本组件实现）

对一条 N 字节 UART 帧 `frame[0..N-1]`（`frame[0]`=地址，`frame[N-1]`=0x6B）：

```
1. CAN 帧类型：扩展帧（29 位 ID）
2. CAN ID     = (addr << 8) | packet,  packet = 0, 1, 2, ...
3. 地址字节 frame[0] 不进 payload，只编码进 CAN ID
4. payload    = frame[1..N-1]（FuncCode + 参数 + 0x6B），每包 8 字节，按 packet 拆分
5. 末包 DLC   = 剩余字节数（可能 < 8）
```

## 例子

### 短命令：UART 帧 `01 36 6B`（读实时位置）

→ **1 包**：

| packet | CAN ID | DLC | data |
|---|---|---|---|
| 0 | 0x0100 | 2 | `36 6B` |

### 长命令：UART 帧 `01 FD 01 0F A0 00 00 01 FA 00 00 00 6B`（13B 位置命令）

地址字节 `01` 不进 payload，剩 12 字节 → 2 包：

| packet | CAN ID | DLC | data |
|---|---|---|---|
| 0 | 0x0100 | 8 | `FD 01 0F A0 00 00 01 FA` |
| 1 | 0x0101 | 4 | `00 00 00 6B` |

### 5.3.1 多电机广播命令（libzdt `zdtBuildMultiMotorCmd`，34B）

地址 `00` 不进 payload，剩 33 字节 → 5 包（8×4 + 1）：

| packet | CAN ID | DLC |
|---|---|---|
| 0 | 0x0000 | 8 |
| 1 | 0x0001 | 8 |
| 2 | 0x0002 | 8 |
| 3 | 0x0003 | 8 |
| 4 | 0x0004 | 1 |

## 真机验证证据

硬件：esp32c3 + SN65HVD230 CAN 收发器 + ZDT_X42S 电机，500 kbps，GPIO7=TX / GPIO6=RX。
对照参考工程 `Documents/PlatformIO/Projects/espidf` 用 UART 给同一电机发 `01 36 6B`
能正常拿到位置数据，证明电机本身识别这条字节序列。

本组件在 CAN 上对同一逻辑命令的两种 payload 编码做对照实验：

| 实验 | CAN ID | payload | 电机回复 |
|---|---|---|---|
| A. PDF 原文（无地址前缀）| 0x0100 | `36 6B` | `36 01 00 00 00 03 6B`（**正常位置数据**）|
| B. 错误勘误（带地址前缀）| 0x0100 | `01 36 6B` | `00 EE 6B`（EE = 格式错误）|
| C. enable PDF 原文 | 0x0100 | `F3 AB 01 00 6B` | timeout（**正常**，enable 异步执行不回）|
| D. enable 错误前缀 | 0x0100 | `01 F3 AB 01 00 6B` | `00 EE 6B`（格式错误）|
| E. 读版本 PDF 原文 | 0x0100 | `1F 6B` | `1F 00 67 23 0A 6B`（**版本号数据**）|

**结论：手册 4.2.1 原文规则正确，地址字节不进 payload，只编码进 CAN ID。**

## 接收（电机返回，手册 4.2.2）

电机返回也用扩展帧，CAN ID = `(Addr<<8)|Packet`，payload = `Code + 状态(02/E2/EE/9F) + 校验码`。
状态码含义：
- `02`：接收的命令正确
- `E2`：参数错误（数据范围/条件不满足）
- `EE`：格式错误
- `9F`：动作执行完成（电机主动返回，如到位/回零完成）

实测：读位置回复 `36 01 00 00 00 03 6B` 中，`36`=功能码回显，`01`=状态（位置正负号），
`00 00 00 03`=位置数据（BE32），`6B`=校验码。
